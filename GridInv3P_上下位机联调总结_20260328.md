# GridInv3P 上下位机联调总结

## 当前状态

- 当前 `GridInv3P` 下位机与 QT 上位机已经完成本轮联调，现阶段可作为稳定备份基线。
- 下位机关键能力已经在既有提交中落地：
  - `c401dd2` `Normalize User_code source encoding to UTF-8`
  - `461f3c8` `Expose inverter PLL debug telemetry over UART`
  - `393498c` `Add inverter telemetry and feedforward tuning hooks`
  - `8ab9033` `Add lower-side inverter reference ramp fallback`
- 也就是说，本轮真正有价值的固件控制逻辑已经在历史提交中，不需要再基于当前工作区的生成物或 IDE 文件做额外备份。

## 当前固件侧已经确认生效的点

### 1. 编码基线

- `User_code` 已统一到 UTF-8。
- 后续继续开发时，直接以 `c401dd2` 之后的版本为准，不需要再回头做 GBK/UTF-8 来回切换。

### 2. PLL 联调抓手

- 固件已经通过 UART 上送并网逆变器调试量，便于 QT 直接判断 PLL 行为：
  - PLL 命令请求/内部使能状态
  - `g_pllLoop.freq`
  - `g_pllLoop.pllLoop.output`
  - `g_freqReport`
  - `g_pllLoop.pllLoop.error`
  - 实际通过校验后的控制字回显

### 3. 下位机参考值 ramp 兜底

- 固件内部已经保留并放慢 `IdRef/IqRef` 的实际生效 ramp。
- 这项改动的意义是：即便后续换别的上位机，或者上位机一次性大步跳变给定，下位机也不会把突变目标直接硬塞进控制环。
- 当前这层 ramp 与 QT 侧软斜坡形成“双层保护”。

### 4. 电压前馈与谐波抑制相关结论

- 现场曾测试过关闭 PR 控制器中的谐波抑制部分，效果没有本质变化，因此问题并不能简单归因到这块。
- 电压前馈系数回到 `0.8` 后，正无功异常场景下的直流母线过压保护触发有所缓和，说明这项参数对动态行为有帮助，应保留当前口径继续作为基线。

## 关键排查结论

### 1. “加功率就 POWER_OK 掉零”不等于协议错误

- 录波已经证明：
  - 上位机命令能下到位
  - 下位机也能正确回读命令
  - PLL 相关状态正常
- 如果这时 `INV3P_POWER_OK_FLAG` 掉零，更应该去看反馈电流、采样、调制裕量和硬件前端，而不是先怀疑串口命令没生效。

### 2. 2026-03-28 这一轮最关键的最终根因是模拟前端未上电

- 当时表现出来的软件侧现象包括：
  - 实际 D/Q 电流几乎为 0
  - PR/电流环输出发散
  - 调制度快速顶满
  - PWM 比较值顶到限幅边界
- 现场最终确认：硬件模拟前端未上电，导致运放与采样链路几乎失效。
- 因此后续任何人接手时，都必须把“先查模拟前端供电”放在前面。

## 当前推荐的带板流程

1. 先确认直流母线供电正常。
2. 确认模拟运放、采样板、电压采样、电流采样链路都已上电。
3. 建立 QT 通信。
4. 先开 PLL，确认锁相正常。
5. 再开功率开关。
6. 给定从小值开始，优先走软斜坡，不做一步大跨。
